## Adapter pattern

All external sources go through adapters that normalize data into the Canonical Event Ledger schema:

```text
SHB API ────────┐
SePay webhook ──┤
KiotViet API ───┤
CSV import ─────┼──► Adapter ──► Canonical Event Ledger (PostgreSQL)
Mini POS ───────┤
Invoice mock ───┘
```

Each adapter implements:

```python
class BaseAdapter(ABC):
    @abstractmethod
    async def fetch(self, merchant_id: str, period: str) -> list[CanonicalRecord]:
        """Fetch data from source and return canonical records."""
    
    @abstractmethod
    async def ingest(self, records: list[CanonicalRecord]) -> IngestionResult:
        """Insert canonical records into ledger. Idempotent."""
```

## SHB Transaction Adapter

| Aspect | Detail |
|---|---|
| Source | SHB transaction sandbox or mock API |
| Auth | API key in header (sandbox) |
| Polling | On-demand per reconciliation run |
| Data mapping | See table below |
| Error handling | Timeout: 10s, retry: 2, fallback: use cached data |
| Idempotency | Deduplicate by `source` + `source_id` |

### SHB to canonical mapping

| SHB field | Canonical field | Notes |
|---|---|---|
| transaction_id | source_id | Prefixed with `SHB-` for canonical ID |
| account_number | merchant_id | Mapped via merchant account lookup |
| amount | amount | Direct |
| counterparty_name | sender_name | Direct |
| counterparty_account | sender_account | Direct |
| description | raw_note | Original Vietnamese text |
| — | normalized_note | Generated by normalization service |
| — | ai_interpretation | Generated by AI (if needed) |
| value_date | transaction_date | Direct |
| transaction_type | transaction_type | Mapped to canonical types |

## SePay Webhook Adapter

| Aspect | Detail |
|---|---|
| Source | SePay sandbox webhook |
| Auth | HMAC-SHA256 signature verification |
| Endpoint | `POST /webhooks/sepay` |
| Processing | Async via Redis queue |
| Idempotency | Deduplicate by SePay transaction ID |
| Error handling | 400 for invalid signature; 200 for valid (even if match fails) |

### SePay webhook to canonical mapping

| SePay field | Canonical field | Notes |
|---|---|---|
| txn_id | source_id | Prefixed with `SEPAY-` |
| amount | amount | Direct |
| sender_name | sender_name | Direct |
| sender_account | sender_account | Direct |
| memo | raw_note | Transfer note / payment reference |
| created_at | transaction_date | Direct |

### Payment reference extraction

The adapter searches the `memo` field for a payment reference pattern (`PAY-[A-Z0-9]{6}`). If found, it triggers exact matching. If not found, the transaction goes through candidate matching.

## LLM Provider

| Aspect | Detail |
|---|---|
| Provider | Abstracted via provider interface (no lock-in) |
| Models | Configurable per agent (e.g., GPT-4 for Planner, GPT-3.5 for note interpretation) |

| Data masking | Sensitive data masked before sending (see `11-security.md` SEC-MASK-001) |
| No training | Provider API configured to not train on KHỚP data |
| Error handling | Timeout: 30s, retry: 1, fallback: rule-based default |

## Configuration

All integration configuration is via environment variables (see `13-environment-setup.md`):

| Env var | Description | Required |
|---|---|---|
| `SHB_API_URL` | SHB sandbox API base URL | Yes |
| `SHB_API_KEY` | SHB sandbox API key | Yes |
| `SEPAY_WEBHOOK_SECRET` | SePay webhook HMAC secret | Yes |
| `SEPAY_API_URL` | SePay sandbox API URL | No |
| `INVOICE_API_URL` | Mock invoice provider URL | Yes |
| `CASE_API_URL` | Mock case management API URL | Yes |
| `LLM_PROVIDER` | LLM provider name | Yes |
| `LLM_API_KEY` | LLM provider API key | Yes |
| `LLM_MODEL_PLANNER` | Model for Planner Agent | Yes |
| `LLM_MODEL_SPECIALIST` | Model for specialist agents | Yes |

## Error handling summary

| Scenario | Behavior |
|---|---|
| SHB API timeout | Retry 2x, then use cached data if available, else log error |
| SePay webhook invalid signature | Return 400, log attempt |
| LLM provider unavailable | Return rule-based default, log error, flag for retry |
| Invoice mock unavailable | Skip invoice check, flag in tax-readiness as "invoice data unavailable" |
| Case API unavailable | Store cases locally, set sync flag for later sync |
